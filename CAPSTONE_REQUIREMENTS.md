##Data Analysis Capstone Requirements##

The Capstone project is your opportunity to bring together all the skills you’ve learned in this program and demonstrate them in a complete, professional-quality data analysis project. This project is designed not only to test your technical ability, but also your reasoning, communication, and professionalism. It should be something you can confidently share as part of your portfolio.

##Core Requirements##

Proper Use of Version Control
Your GitHub repository must include at least 10 separate commits
It's best practice to commit every time you complete a work session to avoid data loss
All commits must be made using the command line
Do not use the file uploader, as this will result in project failure
Data Sources
Your project must include at least two different datasets that can be combined meaningfully.
The combined dataset should have a minimum of 1,000 rows and 10 columns.
All sources must be cited and credited.
Data Analysis
Use Python (Pandas) for data cleaning, wrangling, and exploratory data analysis (EDA).
Demonstrate handling of missing data, duplicates, data types, and transformations.
Database & SQL
Design a relational database and explain your reasoning.
Create and include an Entity-Relationship Diagram (ERD).
Build the database in SQLite3 using Python.
Include at least three intermediate/advanced SQL queries (joins, grouping/aggregation, having, subqueries, etc.).
Functions & Automation
Write at least three unique, custom Python functions that serve a clear purpose or automate repetitive tasks.

##Visualizations##

Include at least three different chart types in your notebook.
Visuals must be well-designed, labeled, and help tell a clear story.
Notebook Quality
Use Markdown cells throughout your Jupyter Notebook to explain your reasoning, analysis, and conclusions. Properly use markdown language to format these cells.
Code should include useful comments.
Notebooks should be well-organized, professional, and free of typos or sloppy formatting. Repetitive or unnecessary code should be deleted.
Repository & README
Your GitHub repository must include:
A polished README.md (professional tone, no errors).
A requirements.txt file.
All data files (or scripts to reproduce them).
Relative paths for all files so the project works across operating systems.

##The README must contain:##

A unique project title (not including words: “capstone” or “Code:You”).
Instructions for setting up, activating/deactivating a virtual environment, and running the project on all operating systems.
Links to all data sources online with credit given and authors cited when possible.
A summary of your main questions and findings (observations and conclusions)
Acknowledgement of any tools, frameworks, or external assistance used - including libraries, frameworks, or AI tools (such as ChatGPT for code generation or editing support).

##Professional Standards##

This project should be written at a professional level - free from grammar errors, typos, and formatting mistakes.
Think of this as something you would hand to a future employer to demonstrate your skills.
You will be interviewed by one or more members of the Code:You team to discuss your project and code.

The project will be graded using the rubric provided.
